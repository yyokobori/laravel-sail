# Config管理仕様（03.06）

更新日: 2026-03-06

## 目的
- 環境設定の参照方法を統一し、実装ブレを防止する。
- 環境差分による不具合を早期検知する。

## 本仕様の対象
- 本仕様は frontend で使用する定数・設定値の管理ルールを定義する。
- Laravel `.env` の機密値は backend 内でのみ使用する。
- frontend が使う値は、公開可能なものだけを build 時に注入（または API 経由で取得）する。

## 基本方針
- 環境変数の直接参照は1箇所に集約する。
- `view/**` / `component/**` / `hooks/**` から環境変数を直接参照しない。
- 設定値は「必須」「任意（デフォルトあり）」を明確に区別する。

## 配置ルール
- 設定の定義・検証・公開は `src/config/**` で管理する。
- 利用側は `src/config` の公開APIのみを参照する。
- 設定キー文字列を各所にハードコードしない。

## 設定値ルール
- 文字列・数値・真偽値は公開時に型変換して返す。
- 数値・URLなど制約がある値は初期化時に検証する。
- 必須値が不足・不正な場合は起動時に検知可能な形で失敗させる。

## デフォルト値ルール
- 任意値のみデフォルト値を持てる。
- デフォルト値は `src/config/**` に集約し、利用側で個別補完しない。
- 環境ごとの差分は値で吸収し、分岐ロジックの増加を避ける。

## セキュリティ/公開範囲
- frontendで公開してよい値のみを設定として扱う。
- 秘密情報（秘密鍵・機密トークン）はfrontend設定に含めない。

## デグレ防止ルール
- 設定キー名の変更は破壊的変更として扱う。
- 設定仕様を変更した場合は、関連文書と `.env` サンプルの整合を確認する。
- 変更時は最低限、型チェックとビルドを通過させる。
