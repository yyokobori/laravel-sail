# API実装仕様（03.04）

更新日: 2026-03-06

## 目的
- API呼び出し実装を統一し、デグレを防止する。
- 引数・戻り値・エラーの扱いを共通化し、画面実装を単純化する。

## 基本方針
- API通信は `api/**` に集約する。
- `view/**` と `component/**` で直接 `fetch` しない。
- データ取得の起点は `view/**` または `hooks/**` に限定する。

## 引数仕様（共通）
- API関数の引数は原則オブジェクト引数を使用する。
- 引数順序は `ai/03.01_arg.md` の優先度に従う。
- boolean引数は禁止する（代替: 文字列リテラル型 / 列挙型）。
- 新規引数追加は optional + default を原則とする。
- 既存必須引数の順序変更・削除は禁止する。

### 引数型サンプル
```ts
type FetchUserParams = {
  userId: string;
  includeArchived?: 'include' | 'exclude';
  styleClasses?: string;
};
```

## 戻り値仕様（共通）
- API関数の戻り値形式は以下に統一する。
  - 成功: `{ ok: true, data: T }`
  - 失敗: `{ ok: false, error: ApiError }`
- 例外を直接 throw して画面側へ伝播させない（内部で `ApiError` へ正規化する）。
- `T` はバックエンド生レスポンス（`response.data`）をそのまま渡す型ではなく、
  アプリ側で利用する整形済み payload 型を指す。

### `T` の扱い（重要）
- 生レスポンス型と画面利用型を分離する。
- `api/**` で生レスポンスを payload へ変換して返す。
- `view/**` は `result.data` を直接利用できる形を前提とする。

### 戻り値型サンプル
```ts
type ApiSuccess<T> = {
  ok: true;
  data: T;
};

type ApiFailure = {
  ok: false;
  error: ApiError;
};

type ApiResult<T> = ApiSuccess<T> | ApiFailure;
```

### 変換方針サンプル
```ts
// raw response -> app payload
type UserResponse = { id: string; user_name: string };
type UserPayload = { id: string; userName: string };

function mapUserResponseToPayload(raw: UserResponse): UserPayload {
  return {
    id: raw.id,
    userName: raw.user_name,
  };
}
```

## エラー仕様（共通）
- エラー型 `ApiError` は最低限以下を持つ。
  - `code`: エラー種別識別子
  - `message`: 表示/ログ用メッセージ
  - `status?`: HTTPステータス
  - `details?`: 項目単位エラー情報（必要時）

### エラー型サンプル
```ts
type ApiError = {
  code: string;
  message: string;
  status?: number;
  details?: Record<string, string[]>;
};
```

## 型配置方針
- API入出力型は `types/**` に配置する。
- API呼び出し関数は `api/**` に配置する。
- 画面表示用の変換が必要な場合は `view/**` または `hooks/**` で行う。

## 実装ルール
- `api/**` はUI描画責務を持たない。
- `component/**` は `api/**` を直接参照しない。
- APIレスポンスとUI状態を混在させない。

## デグレ防止ルール
- 戻り値形式の変更は破壊的変更として扱う。
- 変更時は既存呼び出し箇所を全確認する。
- 最低限、型チェックとビルドを通過させる。

## チェックリスト（変更時）
- [ ] 既存必須引数の順序を変更していない
- [ ] 既存必須引数を削除していない
- [ ] 戻り値形式（ApiResult）を壊していない
- [ ] 失敗時エラーを `ApiError` へ正規化している
- [ ] 呼び出し側の型エラーがない
- [ ] ビルド確認済み
