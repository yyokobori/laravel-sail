# コンポーネント構成仕様（03.03）

更新日: 2026-03-06

## 目的
- コンポーネント構成を統一し、保守性と再利用性を高める。
- import 経路を安定化し、デグレを防止する。

## 基本方針
- コンポーネントは必ず `component/***/` の形式で配置する。
- `component/` 直下にファイルを置かない。
- ディレクトリ単位の一括exportは `index.ts` に統一する。

## コンポーネント設計（必須）
- `component/**` は再利用UI部品のみを担当する。
- `component/**` は props を受け取り、描画する責務に限定する。
- 業務ロジック・画面遷移・API通信の責務は持たない。
- API通信は `api/**` で実施し、`component/**` から直接 `fetch` しない。
- 画面固有ロジックは `view/**`（必要に応じて `hooks/**`）へ分離する。

## 配置ルール（必須）
- 許可:
  - `src/component/form/FormRoot.tsx`
  - `src/component/table/Table.tsx`
  - `src/component/modal/index.ts`
- 禁止:
  - `src/component/Form.tsx`
  - `src/component/index.ts`（全体集約の単一入口は作らない）

## 推奨構成（form例）
```text
src/component/form/
  FormRoot.tsx
  Input.tsx
  ：（略）
  types.ts  # 共通型
  index.ts  # 再export
```

## 公開ルール
- 外部公開は各ディレクトリの `index.ts` から行う。
- 公開名は PascalCase を使用する。
- named export を基本とする。

### index.ts サンプル
```ts
export { FormRoot as Form } from './FormRoot';
export { Input } from './Input';
export { Textarea } from './Textarea';
export { Select } from './Select';
export { Checkbox } from './Checkbox';
export { Radio } from './Radio';

export type { FormProps, BaseFieldProps, StyleClasses } from './types';
```

## importルール
- 利用側は `index.ts` 経由で import する。
- 原則として実体ファイルへの直接 import は行わない（同一ディレクトリ内部を除く）。
- プロジェクト内部 import は `@/` エイリアスで統一する。
- `../../` など階層相対パスによる import は使用しない。

### import例
```ts
import { Form, Input, Select } from '@/component/form';
import type { BaseFieldProps } from '@/component/form';
```

## デグレ防止ルール
- `index.ts` の公開名変更は破壊的変更として扱う。
- 既存公開名は削除せず、必要なら段階的移行（deprecated）を行う。
- 変更時は既存呼び出し箇所を全検索し、ビルド/型チェックを必須とする。
