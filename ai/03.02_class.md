# メインクラス実装規約（確定）

更新日: 2026-03-06

## 目的
- デグレを防ぐ
- 可読性を高める
- 実装の統一性を維持する

## 採用方針
- boolean引数禁止: 採用
- default export制限: 採用（named export を基本）
- hooks配置厳格化: 採用

## 命名規則
- 命名規則は `ai/03_react.md` に集約する。
- 本書では命名規則の重複定義を行わない。

## 1. 引数規約
- 引数の共通仕様は `ai/03.01_arg.md` を参照する。
- 引数が3つ以上になる場合はオブジェクト引数を使用する。
- 既存必須引数の順序は変更しない。
- 新規引数は optional + default を原則とする。
- boolean引数は禁止する。
  - 代替: 文字列リテラル型 / 列挙型を使用する。

## 2. export規約
- named export を基本とする。
- default export は原則使用しない。
- 例外を認める場合は、事前に明示ルールを追加する。

## 3. hooks配置規約
- hooks はコンポーネント関数の先頭で呼び出す。
- 条件分岐・ループ・ネスト関数内で hooks を呼ばない。
- hooks の呼び出し順は固定し、途中で分岐しない。
- カスタムhooksは `useXxx` 命名を必須とする。

## 4. import順
1) 外部ライブラリ
2) 外部拡張（router等）
3) プロジェクト内部import（`@` エイリアス）
4) スタイルimport

補足:
- `../../` のような階層相対importは使用しない。
- プロジェクト内部参照は `@/` 始まりで統一する。

## 5. 記述順（ファイル内）
1) import
2) 型定義
3) 定数
4) 補助関数
5) メインクラス（メインコンポーネント）
6) named export

## 6. 既存呼び出し影響チェック
- チェック項目は `ai/03.01_arg.md` の「既存呼び出し影響チェック（必須）」を参照する。
