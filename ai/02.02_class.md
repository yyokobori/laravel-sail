# Laravelクラス実装規約（案）

更新日: 2026-03-10

## 目的
- デグレ防止・保守性向上・実装統一

## 採用方針
- 型宣言・型ヒント必須（PHP 8+ 準拠）
- PSR-12コーディング規約準拠
- DI（依存性注入）を優先し、new禁止
- ファサード利用は最小限

## 命名規則
- クラス名・ファイル名はPSR-4に準拠（StudlyCase）
- サービス/ユーティリティは「XxxService」「XxxUtil」命名
- コントローラは「XxxController」
- ミドルウェアは「XxxMiddleware」
- プロバイダは「XxxServiceProvider」

## 1. 引数・戻り値規約
- すべての引数・戻り値に型宣言を付与
- 配列は型付きコレクション（array<int, User>等）をDocBlockで明示
- オプション引数はデフォルト値を明示
- boolean引数は極力避け、enum/定数で代替

## 2. export/アクセス修飾子規約
- クラス・メソッド・プロパティは適切なアクセス修飾子（public/protected/private）を必ず明示
- 不要なpublicは避ける

## 3. DI/依存性注入規約
- サービス・リポジトリ・コンテキスト等はコンストラクタインジェクションを原則とする
- newによる直接生成は禁止（例外：値オブジェクト等）

## 4. import/use順
1) PHP標準ライブラリ
2) フレームワーク（Illuminate等）
3) サードパーティ
4) アプリケーション内部（App\）
5) function/const use

## 5. ファイル内記述順
1) namespace宣言
2) use宣言
3) クラスDocBlock
4) クラス定義
5) プロパティ
6) コンストラクタ
7) publicメソッド
8) protected/privateメソッド
9) 補助関数

## 6. 既存呼び出し影響チェック
- 既存のサービス/コントローラ/テスト等の呼び出し箇所を必ずgrep等で確認し、影響範囲を明示

## 7. クラス・メソッドのコメント規約（phpdoc）

- すべてのクラス・publicメソッドにはphpdocコメントを必須とする
- クラスphpdocには「責務」「利用例（必要に応じて）」を簡潔に記載
- メソッドphpdocには「引数」「戻り値」「例外」「副作用」を明記
- 型情報はphpdocと型宣言の両方で冗長に記載してもよい
- プロパティにも必要に応じてphpdocを付与

### クラスphpdoc例
```php
/**
 * ユーザー情報を管理するサービス
 *
 * @package App\Services
 */
class UserService
{
    // ...
}
```

### メソッドphpdoc例
```php
/**
 * ユーザーをIDで取得する
 *
 * @param int $id ユーザーID
 * @return User|null
 * @throws UserNotFoundException
 */
public function findById(int $id): ?User
{
    // ...
}
```

### プロパティphpdoc例
```php
/** @var int|null ユーザーID */
private ?int $userId = null;
```

## 8. メソッド内コメント規約

- 変数定義時は、用途や型・値の意味を1行コメントで必ず記載する
- 処理の大枠（主要な処理ブロックの開始前）には、何をしているかを明記する
- 分岐（if/else/switch）やループ（for/foreach/while）の条件・意図は必ずコメントで補足する
- 複雑な処理や副作用がある箇所は、なぜその処理が必要かも記載する
- コメントは日本語で簡潔に書く（英語コメントは原則禁止、外部API仕様等は除く）

### サンプル

```php
public function processOrder(array $items): bool
{
    // 注文合計金額
    $total = 0;

    // 商品ごとに金額を加算
    foreach ($items as $item) {
        // 在庫数が0の場合はスキップ
        if ($item['stock'] === 0) {
            continue; // 在庫切れ
        }
        $total += $item['price'] * $item['quantity'];
    }

    // 合計金額が0円の場合はエラー
    if ($total === 0) {
        // 商品が1つも有効でなかった場合
        return false;
    }

    // 決済処理（外部API呼び出し）
    // ...（省略）

    return true;
}
```
