# 02.11_dependency.md

## 概要

本仕様は、Laravelバックエンドにおける依存関係（Dependency）のルールを定義する。

目的：

- レイヤー間の責務分離を維持する
- 循環依存を防止する
- AI実装時の依存関係のブレ防止
- テスト容易性の向上

---

## 基本方針（最重要）

- 依存は上位レイヤーから下位レイヤーへのみ許可する
- 逆方向の依存は禁止
- 依存はコンストラクタインジェクションで行う
- インターフェースを介して依存する
- newによるインスタンス生成は禁止

---

## 依存関係ルール

### 許可される依存

Controller → Service  
Service → Repository  
Repository → Model  

---

### 禁止される依存

- Service → Controller
- Repository → Service
- Model → Repository
- Model → Service
- Controller → Repository（直接アクセス禁止）

---

## 依存方向イメージ

Controller  
  ↓  
Service  
  ↓  
Repository  
  ↓  
Model  

---

## インターフェース利用

### 基本ルール

- ServiceはRepositoryのInterfaceに依存する
- 実装クラスには直接依存しない

---

### 例

interface UserRepositoryInterface
{
    public function create(array $data);
}

class UserRepository implements UserRepositoryInterface
{
}

class UserService
{
    private UserRepositoryInterface $userRepository;

    public function __construct(UserRepositoryInterface $userRepository)
    {
        $this->userRepository = $userRepository;
    }
}

---

## バインディング

### 定義場所

- App\Providers\AppServiceProvider

---

### 例

public function register()
{
    $this->app->bind(
        UserRepositoryInterface::class,
        UserRepository::class
    );
}

---

## コンストラクタインジェクション

### 必須ルール

- すべての依存はコンストラクタで受け取る
- メソッドインジェクションは禁止
- Facade依存は最小限にする

---

### NG例

$repo = new UserRepository();

---

## Service間依存

### 方針

- 原則禁止
- 必要な場合はFacadeやUtilではなく、設計を見直す

---

### 理由

- トランザクション管理が崩れる
- 責務が曖昧になる
- 循環依存の原因になる

---

## Repositoryルール補足

- Repositoryは他のRepositoryに依存しない
- 複数テーブル操作はServiceで統合する

---

## Model依存

- Modelはどのレイヤーにも依存しない
- 単なるデータ構造として扱う

---

## Utilクラス

### 使用ルール

- 副作用のない処理のみ許可
- Serviceの補助として使用する
- DBアクセスは禁止

---

## DIコンテナ

- LaravelのService Containerを使用する
- 自動解決を前提とする
- 手動生成は禁止

---

## テスト容易性

- すべての依存はモック可能にする
- Interfaceを利用することで差し替え可能にする

---

## 禁止事項

- newによる依存生成
- レイヤー逆依存
- Repositoryの直接呼び出し（Controllerから）
- Serviceの多重依存
- 静的クラスへの過度な依存

---

## AI向け実装指針

- 依存は必ずInterface経由で定義する
- コンストラクタで受け取る
- bind定義を忘れない
- レイヤー構造を破らない
- 不要な依存を追加しない
