# ■ 02.09_logging.md（追加・強化ルール）

## ■ ログフォーマット

### ● JSON形式（必須）

すべてのログは**JSON形式で出力する**

例：

```json
{
  "level": "info",
  "message": "User created",
  "timestamp": "2026-03-27T12:00:00+09:00",
  "request_id": "xxx",
  "user_id": 1,
  "route": "user.store",
  "context": {
    "user_id": 1
  }
}
```

---

### ● 必須フィールド

| 項目         | 内容        |
| ---------- | --------- |
| level      | ログレベル     |
| message    | イベント内容    |
| timestamp  | ISO8601   |
| request_id | リクエスト単位ID |
| context    | 可変データ     |

---

## ■ SQLログルール（新規）

### ● 基本方針

* SQLログは**Serviceの基底クラスで一元管理する**
* Repositoryでは出力しない（責務分離）
* LaravelのQueryLog / DB::listen を利用する

---

### ● 出力条件

| 環境    | 出力 |
| ----- | -- |
| local | 〇  |
| stg   | ×  |
| prd   | ×  |

---

### ● 出力内容

* SQLクエリ
* バインドパラメータ
* 実行時間（ms）

例：

```json
{
  "level": "debug",
  "message": "SQL executed",
  "sql": "select * from users where id = ?",
  "bindings": [1],
  "time_ms": 12
}
```

---

### ● 実装方針

* BaseService で DB::listen を登録
* 環境判定で制御

---

## ■ エラーログ詳細（強化）

エラーログには以下を必ず含める：

| 項目         | 内容         |
| ---------- | ---------- |
| error_code | 定義済みエラーコード |
| exception  | 例外クラス名     |
| message    | エラーメッセージ   |
| file       | 発生ファイル     |
| line       | 行番号        |

例：

```json
{
  "level": "error",
  "message": "User creation failed",
  "error_code": "E500001",
  "exception": "App\\Exceptions\\UserCreateException",
  "file": "UserService.php",
  "line": 120
}
```

# ■ パフォーマンスログ（設計＋ロジック）

---

## ■ 基本方針

* **Service単位で計測する**
* すべての業務処理に対して記録する
* JSONログに `duration_ms` を追加

---

## ■ 出力イメージ

```json
{
  "level": "info",
  "message": "UserService.create",
  "duration_ms": 42,
  "status": "success"
}
```

---

## ■ 実装パターン（推奨）

### ● BaseServiceで共通化

```php
abstract class BaseService
{
    protected function executeWithLog(string $event, callable $callback)
    {
        $start = microtime(true);

        try {
            $result = $callback();

            $this->logPerformance($event, $start, 'success');

            return $result;

        } catch (\Throwable $e) {

            $this->logPerformance($event, $start, 'error');

            throw $e;
        }
    }

    private function logPerformance(string $event, float $start, string $status): void
    {
        $duration = (microtime(true) - $start) * 1000;

        \Log::info('performance', [
            'event' => $event,
            'duration_ms' => (int)$duration,
            'status' => $status,
        ]);
    }
}
```

---

## ■ Service側の使い方

```php
public function createUser(array $data)
{
    return $this->executeWithLog('user.create', function () use ($data) {
        return $this->userRepository->create($data);
    });
}
```

---

## ■ ポイント

* ロジックに侵入しない（ラップするだけ）
* 全Serviceで強制適用できる
* 将来APM（Datadog）にも繋げやすいようにする

# ■ セキュリティログ

---

## ■ 基本方針

* 専用ログ（security.log）に出力
* 「攻撃・不正検知」を記録する
* infoログとは分離

---

## ■ 対象イベント

| 種類     | 例           |
| ------ | ----------- |
| 認証     | ログイン成功 / 失敗 |
| 認可     | 403エラー      |
| 不正アクセス | トークン不正      |
| 入力異常   | バリデーション異常多発 |
| レート制限  | throttle    |

---

## ■ 出力例

```json
{
  "level": "warning",
  "message": "login.failed",
  "user_id": null,
  "ip": "192.168.0.1",
  "email": "masked",
  "reason": "invalid_password"
}
```

---

## ■ 実装方針

### ● 専用チャネル

```php
'security' => [
    'driver' => 'daily',
    'path' => storage_path('logs/security.log'),
    'level' => 'info',
],
```

---

### ● 専用クラス

```php
class SecurityLogger
{
    public static function log(string $event, array $context = [])
    {
        \Log::channel('security')->warning($event, $context);
    }
}
```

---

## ■ 使い方

```php
SecurityLogger::log('login.failed', [
    'email' => $email,
    'ip' => request()->ip(),
]);
```

---

## ■ ポイント

* 将来SIEM（セキュリティ監視）に流せる
* CloudWatchでも分離される

# ■ エラークラス設計

---

## ■ 基本方針

* **業務例外とシステム例外を分ける**
* Errorコードと1対1対応させる
* Serviceからthrowする

---

## ■ ベースクラス

```php
abstract class BaseException extends \Exception
{
    protected string $errorCode;
    protected int $httpStatus;

    public function getErrorCode(): string
    {
        return $this->errorCode;
    }

    public function getHttpStatus(): int
    {
        return $this->httpStatus;
    }
}
```

---

## ■ 種類

### BusinessException（業務エラー）

```php
class BusinessException extends BaseException
{
}
```

用途：

* バリデーション以外の業務NG
* 例：重複登録、状態不正

---

### NotFoundException

```php
class NotFoundException extends BaseException
{
}
```

用途：

* データなし（404）

---

### UnauthorizedException

```php
class UnauthorizedException extends BaseException
{
}
```

用途：

* 認証エラー（401）

---

### ForbiddenException

```php
class ForbiddenException extends BaseException
{
}
```

用途：

* 権限エラー（403）

---

### SystemException（重要）

```php
class SystemException extends BaseException
{
}
```

用途：

* 想定外エラー
* DBエラー
* 外部API障害
* **error / critical対象**

---

### ValidationException（Laravel標準）

---

これはそのまま使う（ルール通り）

## ■ おすすめ追加

### ● ConflictException

```php
class ConflictException extends BaseException
{
}
```

用途：

* 409系（重複・競合）

---

### ● ExternalServiceException

```php
class ExternalServiceException extends SystemException
{
}
```

用途：

* 外部API失敗

---

## ■ Handler側の扱い

* Business系 → 4xx
* System系 → 5xx
* ログレベル変える

---


## ■ 加止事項（強化）

* SQLログを本番に出力する
* Exceptionを文字列だけで記録する（構造化しない）
* contextなしログを出力する
* JSONフォーマットを崩す
