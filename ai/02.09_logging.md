# 02.08_error.md

## 概要

本仕様は、Laravelバックエンドにおけるエラーハンドリングおよび例外設計ルールを定義する。

目的：

- 例外処理の一元管理
- APIレスポンスの統一
- AI実装時のブレ防止
- 障害時の調査性向上

---

## 基本方針（最重要）

- 例外はすべてthrowする
- try-catchで握り潰さない
- 例外処理はHandlerに集約する
- レスポンスは必ず統一フォーマットに従う
- OpenAPI仕様と完全一致させる

---

## レイヤー責務

| レイヤー | 例外処理 |
|----------|----------|
| Controller | 禁止 |
| Request | バリデーションのみ |
| Service | throwのみ |
| Repository | 原則禁止 |
| Model | 禁止 |
| Resource | 禁止 |
| Handler | 一元処理 |

---

## エラー分類

### 業務エラー（Business）

- 想定内のエラー
- HTTP 4xx

例：
- データ不正
- 状態不整合
- 重複登録

---

### システムエラー（System）

- 想定外のエラー
- HTTP 5xx

例：
- DB接続失敗
- 外部API障害
- 未捕捉例外

---

## エラーコード仕様

### フォーマット

E + HTTPステータス + 3桁連番

例：

- E400000
- E404001
- E500000

---

### 定義場所

app/Constants/Errors.php

---

### 使用ルール

- すべての例外に付与する
- レスポンスに含める
- ログにも出力する

---

## レスポンス仕様

### 基本構造

{
  "error": {
    "code": "E400000",
    "message": "入力内容が不正です"
  }
}

---

### ポリシー

- messageはユーザー向け
- 詳細情報は返さない
- 内部情報はログにのみ出力する

---

## 例外クラス設計

### BaseException

abstract class BaseException extends \Exception
{
    protected string $errorCode;
    protected int $httpStatus;

    public function getErrorCode(): string
    {
        return $this->errorCode;
    }

    public function getHttpStatus(): int
    {
        return $this->httpStatus;
    }
}

---

### BusinessException

class BusinessException extends BaseException
{
}

用途：
- 業務ロジックエラー

---

### NotFoundException

class NotFoundException extends BaseException
{
}

用途：
- データ未存在

---

### UnauthorizedException

class UnauthorizedException extends BaseException
{
}

用途：
- 認証エラー

---

### ForbiddenException

class ForbiddenException extends BaseException
{
}

用途：
- 権限エラー

---

### ConflictException

class ConflictException extends BaseException
{
}

用途：
- 重複・競合

---

### SystemException

class SystemException extends BaseException
{
}

用途：
- 想定外エラー

---

### ExternalServiceException

class ExternalServiceException extends SystemException
{
}

用途：
- 外部APIエラー

---

## Handlerルール

- すべての例外をここでキャッチする
- BaseExceptionは個別処理
- その他はSystemExceptionとして扱う

---

## ログ出力

- error以上でログ出力
- error_codeを必ず含める

例：

Log::error('User creation failed', [
    'error_code' => 'E500001',
]);

---

## 禁止事項

- try-catchで握り潰す
- 詳細エラーをレスポンスに含める
- error_code未設定の例外を投げる
- レイヤー外で例外処理を行う

---

## AI向け実装指針

- 例外は必ずthrowする
- returnでエラーを返さない
- Serviceで例外を生成する
- Handlerでレスポンス変換する
- エラーコードは必ず定義済みを使用する
