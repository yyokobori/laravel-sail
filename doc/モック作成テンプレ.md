# 目的
指定した Laravel Service Interface のモックテストを Mockery を使って作成してください。
Request バリデーションも確認するテストを含めてください。

# 入力
- Service Interface: [対象の Interface 名]
- メソッド名: [テスト対象メソッド]
- 引数サンプル: [引数データ例]
- 戻り値サンプル: [Resource / 配列など]

# 出力条件
- PHPUnit テストクラスで作成
- Mockery を使用して Service のモックを作成
- tearDown で Mockery::close() を呼ぶ
- Request Validator もテストに含める
- コメントは最小限でコードがコピー可能
- 推測値は使わず、提供データのみを利用すること
- テストの目的が明確にわかるメソッド名にすること

# 注意
- 実際の DB にはアクセスしない
- 他の Service はすべてモックに置き換える
- OpenAPI / Resource の仕様に基づき戻り値を作る
