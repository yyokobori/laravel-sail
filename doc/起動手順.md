# Laravel 開発環境構築手順（Windows + WSL2 + Docker Desktop + VSCode Remote WSL）

このドキュメントでは、WSL2 + Docker Desktop を用い、Laravel Sail で PHP をホストにインストールせずに開発環境を構築します。

## ディレクトリ構成
```
~/work/sail-app/
  ├ doc/          # ドキュメント（本ファイル）
  ├ ai/           # AI用情報共有ファイル
  ├ app/          # Laravel プロジェクト本体
  └ sh/           # 起動・セットアップ用シェル
       ├ setup.sh
       ├ start.sh
       ├ stop.sh
       └ restart.sh
```

## 1. 初期セットアップ（1 回のみ）
以下を実行してアプリを生成し、Sail 設定を追加します。

```bash
cd ~/work/sail-app
./sh/setup.sh
```

`setup.sh` の処理:
1. `app/` が空なら Laravel プロジェクトを composer（Docker 版）で作成
2. `laravel/sail` を dev 依存で追加
3. `php artisan sail:install --with=mysql` を Docker コンテナ内で実行し、`compose.yaml` を生成
4. 必要に応じて `.env.example` をコピー

完了時には `app` 以下に Laravel ファイル群と `compose.yaml`、`.env` が生成されます。

## 2. 起動
```bash
cd ~/work/sail-app
./sh/start.sh
```

`start.sh` の処理:
- 存在しない場合、`.env.example` を `.env` にコピーする
- Sail コンテナを `up -d` で起動
- アプリケーションキーを生成
- マイグレーションを実行

起動後は `http://localhost` でウェルカムページを確認できます。

## 3. 停止／再起動
```
./sh/stop.sh      # コンテナを全停止
./sh/restart.sh   # 再起動（down→up）
```

## 4. ローカル設定ファイルの取り扱い

`start.sh` 実行時、プロジェクトルート直下の `local/` フォルダに置かれた
設定ファイルが `app/` 以下へ自動コピーされます。

```
local/
  ├ .env.example       # アプリ用環境変数テンプレート
  ├ mysql.cnf          # MySQL 設定
  └ php.ini            # PHP 設定
```

- コピー対象は `.env.example`, `mysql.cnf`, `php.ini`。
- すべてのファイルは `app/` 配下に上書きされる。
- コピー先のファイルがコピー元より**新しいタイムスタンプ**の場合は
  警告を出して処理を中止し、Sail を起動しません。

この仕組みにより、環境固有の設定をバージョン管理対象外で保持できます。

## 5. Docker Compose の拡張
`app/compose.yaml` はデフォルトで `laravel.test` と `mysql` を含みます。
以下を追加してください（`setup.sh` 実行後に手動で編集するか、ここに反映しておく）。

```yaml
    # S3 / MinIO サービス
    s3:
        image: minio/minio
        command: server /data
        ports:
            - '9000:9000'
        environment:
            MINIO_ROOT_USER: '${S3_ACCESS_KEY}'
            MINIO_ROOT_PASSWORD: '${S3_SECRET_KEY}'
        volumes:
            - 'sail-minio:/data'
        networks:
            - sail

    # API 仕様表示 (Swagger UI)
    api-docs:
        image: swaggerapi/swagger-ui
        ports:
            - '8080:8080'
        environment:
            SWAGGER_JSON: /openapi.yaml
        volumes:
            - './app/docs/openapi.yaml:/openapi.yaml'
        networks:
            - sail

volumes:
    sail-minio:
        driver: local
```

## 5. VSCode からの利用
1. VSCode Remote - WSL で `~/work/sail-app` を開く
2. ターミナルで `./sh/setup.sh`（初回）または `./sh/start.sh` を実行
3. ブラウザを開き `http://localhost`

## 6. よくあるトラブル
- `Docker is not running` → Docker Desktop を起動
- `/env` コピーエラー → `chmod +x sh/*.sh` で実行権限付与
- 権限エラー → `chown -R $(id -u):$(id -g) app`
- MySQL 接続失敗 → `.env` の DB_HOST, USER, PASS を確認

その他は `app/` 以下のログ (`storage/logs/laravel.log`) を参照。
