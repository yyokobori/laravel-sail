# 設定ファイルについて

このプロジェクトでは、ローカル環境固有の設定を `conf/local/` ディレクトリに集約し、
`start.sh` 実行時に自動的に Laravel アプリ本体へ反映させる仕組みを採用しています。

## conf/local ディレクトリの構成
ルート (`sail-app/`) の `conf/` 配下に以下のようなファイルを配置します。

```
conf/
  ├ local/                 # ローカル開発環境用設定
  │  ├ .env.example        # ベースの環境変数テンプレート
  │  ├ mysql.cnf           # MySQL 用カスタム設定
  │  └ php.ini             # PHP 用カスタム設定
  └ aws/                   # AWS環境用設定（将来追加予定）
```

- これらのファイルはコミットせず、個人や環境ごとの内容を記述します。
- `start.sh` は存在するファイルを検知し、自動で `app/` 以下へコピーします。

## コピーのルール
1. `.env.example` は `app/.env` へコピーされ、既存の `.env` があれば上書きされます。
2. `mysql.cnf` と `php.ini` はそれぞれ `app/` に配置されます。
3. コピー元ファイルよりコピー先のタイムスタンプが新しい場合は**警告が表示され**、
   処理を中断して Sail の起動を行いません。誤って上書きしたくないときに役立ちます。

## 手動での確認・修正
- 必要に応じて `conf/local/` 内のファイルを編集し、`./sh/start.sh` を再実行してください。
- `app/` 側で直接編集した場合は、その変更が上書きされる可能性があります。
  上書き禁止の意図がある場合は `conf/local/` のファイルを更新するか、
  起動前に `app/` のタイムスタンプを確認してください。

## その他の設定ファイル
- 将来的に別の設定ファイルを反映させたい場合は、`start.sh` のコピーリストに追加します。
- 例えば `conf/local/php-fpm.conf` や `conf/local/supervisord.conf` なども同様に扱えます。
- AWS 環境用の設定は `conf/aws/` に配置し、別途スクリプトで参照するようにします。
